## Design System Generation (v2.0 Feature)

For ANY project, first generate a design system:

### Process

```
1. USER REQUEST
   "Build a landing page for my beauty spa"

2. DOMAIN SEARCH (5 parallel)
   - Product type → style matching
   - UI style recommendations
   - Color palette selection
   - Landing page patterns
   - Typography pairing

3. REASONING ENGINE
   - Match product → UI category rules
   - Filter anti-patterns for industry
   - Apply decision rules

4. COMPLETE DESIGN SYSTEM
   - Pattern + Style + Colors + Typography
   - Effects + Anti-patterns
   - Pre-delivery checklist
```

## Color System

### 60-30-10 Principle

| Role | % | Purpose |
|------|---|---------|
| Dominant | 60% | Background |
| Secondary | 30% | Supporting elements |
| Accent | 10% | CTA, highlights |

### Anti-Patterns to AVOID

| Industry | Never Use |
|----------|-----------|
| Banking | Neon colors, AI purple/pink gradients |
| Healthcare | Bright neon, harsh animations |
| Finance | Dark mode only, complex charts |
| Luxury | AI-generated generic styles |

## Pre-Delivery Checklist

Before delivering ANY UI:

- [ ] **No emojis as icons** - Use SVG (Heroicons, Lucide)
- [ ] cursor-pointer on clickables
- [ ] Hover states (150-300ms smooth)
- [ ] Text contrast 4.5:1 minimum
- [ ] Focus states for keyboard nav
- [ ] prefers-reduced-motion
- [ ] Responsive: 375px, 768px, 1024px, 1440px
